Project Idea

Overview

My project idea involves creating a game centred around cars and automobiles in general. It would involve high levels of detail surrounding creation and assembly of vehicles, such as in-depth creation and tuning of engines of different types, as well as in-depth creation and tuning of suspension parts, and then allow you to test your car and race it against others. By having many parts available to the user, they are able to create and modify a vehicle to make it unique to themselves not only including the exterior appearance of the vehicle, but also the internal functionality such as high-performance engines, drift, race, drag suspension etc.

Motivation

Currently as of March 2019, there are only 2 games that fit this model, one being released in 2003, called Street Legal Racing Redline. Although still popular, the game is extremely outdated and runs poorly on modern hardware. The other game that fits this model is called Street Tuning Evolution, which was first brought into conception in 2015, but has yet to be completed and still has not reached alpha stages. There is a huge player-base for the original 2003 game, yet no major game studio and publisher have attempted to break into this market. With modern technology, a modern version of this game would satisfy the demands of many users.

Description

The game would make use of many individual parts of vehicles in order to allow the user to build their vehicle. Parts would be split up into separate groups for different sections of the car, such as a category for engine parts, which would be further categorised into sub-categories such as engine blocks, engine heads, fuel delivery, air delivery etc, which would each contain parts relating to that group, for example, the air delivery category would have parts such as carburettors, air filters, intake manifolds etc, while a performance parts category would host parts such as turbochargers, superchargers, nitrous oxide systems etc.

The one game that is currently on the market, SLRR, only has one way to earn money, which is street-racing. This limits gameplay and can make the game become boring and repetitive after time, causing users to stop playing. My project idea would incorporate many different race types, such as official racing, eg circuit racing on official tracks such as Bathurst or Spa, along with drift type events, as well as street racing, which allows users the opportunity to customise a car to compete in the events that they prefer, and also gives motive for the user to continue playing, so as to develop 3 separate cars for each different racing discipline.

The original game also did not include any form of multiplayer, meaning the users were stuck to only playing against AI. By incorporating multiplayer into my project, this opens up a range of new features that could be implemented into the game, such as having free roam amongst other users to look at each-others cars, and race one another or even host events such as a circuit race, drift events, or street races, which would give the users the ability to compete with others to see who has built the fastest machine. This further increases motive to play so as to create a vehicle that is faster than everyone else's.

A system of buying pre-owned vehicles would need to be set-up, which would be at the start of the game and makes the suer start off with a slow car, before being able to race and buy performance parts to win more races and earn more money. After this, the user may use the online multiplayer feature to use an auction house, which would allow users to put their cars on auction and buy other players vehicles, potentially at discounted prices if the vehicles have been written off or are damaged.

The game would also employ a full damage model, in order to simulate engine wear and tear, as well as body parts crumpling from crashes and suspension components being destroyed. This means the user will need to be careful when racing in order to preserve the condition of their car to continue racing it, and means users cannot just put a turbocharger on a vehicle and turn the boost up to the maximum possible, as this will destroy the engine and set the user back.

Tools & Technologies

In order to produce this game, a game engine would be needed. My project would most likely make use of the Unity engine due to it being free, however, it may need to be modified in order to accommodate some of the features of my project, such as a damage model and multiplayer. A modelling software would also need to be used to create the models of every engine, suspension and vehicle component available, as well as the vehicles themselves. This could be done using Blender, as it is also free software and easy to use.

Skills Required

Knowledge on programming and game development will be essential in order to create the game, along with knowledge and experience with modelling, as each individual part of the engine, suspension and chassis will need to be modelled. The computers used to create these models would also need to be high-end machines in order to render models in a timely manner (eg high end consumer i7 processors, 6+ cores) The feasibility of gathering all the talent is high, however, many people with these skills would need to be employed in order to keep production time low, this coupled with the need for several high-end desktops for programming and rendering models would drive costs up very high, meaning a large amount of money would be needed to fund this project.

Outcome

If this project were to be successful, many people would be able to enjoy a modern equivalent of the 2003 SLRR, which is still widely played today due to its high customisation of vehicles, thereby solving the problem of having to play the outdated game. This will potentially have a large impact on not only the gaming industry, but also car enthusiasts, as they will be able to visualise upgrades to their vehicles in the game before they purchase them for their real car, allowing them to see if they like it before-hand.
